# pasport-queue-watcher

Стежить за сторінкою електронної черги **pasport.org.ua** і надсилає сповіщення
в **Telegram**, коли зʼявляються вільні місця для запису. Додатково повідомляє
про стан **самого автомата**: «я живий», «не можу перевірити», «знову працює».

За замовчуванням стежить за:
`https://wroclaw.pasport.org.ua/solutions/e-queue`

Працює у хмарі через **GitHub Actions** (нічого не треба тримати ввімкненим),
або локально на своєму компʼютері.

---

## Як воно розпізнає місця

Сторінки `pasport.org.ua` рендеряться **на сервері** (SSR) — потрібний текст уже є
у HTML, тож достатньо простого HTTP-запиту (`USE_BROWSER=0`, за замовчуванням).
Браузер (Playwright) не потрібен.

Скрипт розрізняє **три** стани сторінки:

| Стан | Як визначає (підтверджено 07.2026) | Дія |
|------|-------------------------------------|-----|
| 🟢 **є місця** | видно форму запису (`queueForm` / `qlogickForm`) і немає фрази «зайнято» | сповіщення «зʼявилися місця» |
| 🔴 **місць немає** | є фраза **«Наразі всі місця зайняті»** і немає форми | нічого |
| ⚠️ **помилка** | Cloudflare-челендж («Just a moment…»), сторінка не завантажилась або виняток | сповіщення про несправність (див. нижче) |

Логіка навмисне зміщена «не пропустити місце»: якщо немає ні «зайнято», ні форми
(нетипова сторінка) — вважає, що місця **можуть** бути, і шле сповіщення з позначкою
«перевір вручну». Дефолтні фрази/маркери вже підлаштовані під реальний сайт;
змінювати їх зазвичай **не треба**.

> 💡 **Ще надійніший варіант** (на майбутнє). Якщо сайт почне вантажити слоти окремим
> запитом до API — відкрий DevTools (F12) → **Network**, знайди запит із JSON і
> моніторь його напряму. Наразі SSR-розпізнавання за текстом працює.

## Сповіщення про стан автомата («працює / не працює»)

Окрім «є місця», бот повідомляє, що з ним самим:

- **⚠️ «Не можу перевірити сторінку»** — коли `ERROR_ALERT_AFTER` перевірок поспіль
  (типово 3 ≈ 15 хв) зазнали невдачі (сайт лежить, Cloudflare блокує, помилка).
  Надсилається **один раз**, коли ламається (без спаму).
- **✅ «Автомат знову працює»** — коли після збою перевірка знову вдалась.
- **🤖 «Автомат працює»** — періодичний heartbeat раз на `HEARTBEAT_HOURS` годин
  (типово 24) зі зведенням стану кожної сторінки. `HEARTBEAT_HOURS=0` — вимкнути.
  Перше сповіщення приходить одразу після першого запуску — так ти бачиш, що все ожило.

> ℹ️ Ці сповіщення (і антиспам, і heartbeat, і виявлення «зʼявилися місця») спираються
> на файл `state.json` між запусками. На GitHub Actions він зберігається через кеш
> (крок *Restore state*). Якщо кеш загубиться — можеш отримати повторне «Автомат
> запущено» чи зайве нагадування; це не помилка.

## Частина 1. Створюємо Telegram-бота

1. У Telegram напиши боту **@BotFather** → команда `/newbot` → придумай імʼя.
   Отримаєш **токен** виду `123456789:AAE...` — це `TELEGRAM_BOT_TOKEN`.
2. Напиши щось **своєму новому боту** (будь-яке повідомлення, напр. «привіт») —
   інакше він не зможе тобі писати.
3. Дізнайся свій **chat_id**: напиши боту **@userinfobot** — він одразу пришле
   твій `Id`. Це `TELEGRAM_CHAT_ID` (число, іноді зі знаком «-» для груп).

## Частина 2. Запуск на GitHub Actions (рекомендовано)

### 2.1. Залий проєкт у свій репозиторій

Розпакуй архів, відкрий термінал у папці `pasport-queue-watcher` і виконай:

```bash
git init
git add .
git commit -m "pasport e-queue watcher"
git branch -M main
git remote add origin https://github.com/IgorKalch/pasport-queue-watcher.git
git push -u origin main
```

> Спершу створи порожній репозиторій `pasport-queue-watcher` на GitHub
> (без README/ліцензії, щоб не було конфлікту). Або скористайся `gh`:
> `gh repo create IgorKalch/pasport-queue-watcher --public --source=. --push`

> 🔸 **Публічний чи приватний?** У **публічних** репозиторіях хвилини GitHub Actions
> безкоштовні й безлімітні. У **приватних** — 2000 хв/міс на безкоштовному тарифі.
> За замовчуванням `USE_BROWSER=0` (простий HTTP, без браузера), тож прогін триває
> секунди і хвилини майже не витрачаються. Якщо все ж увімкнеш `USE_BROWSER=1` —
> кожен прогін довший (встановлення Chromium), тоді для приватного репозиторію
> зроби інтервал рідшим (`*/15`).

### 2.2. Додай секрети

Репозиторій → **Settings → Secrets and variables → Actions → вкладка Secrets →
New repository secret**. Додай два:

| Ім'я | Значення |
|------|----------|
| `TELEGRAM_BOT_TOKEN` | токен від BotFather |
| `TELEGRAM_CHAT_ID`   | твій chat_id |

### 2.3. (Необовʼязково) Додай змінні

Там само, вкладка **Variables → New repository variable**. Усі — опційні:

| Ім'я | Приклад | Навіщо |
|------|---------|--------|
| `WATCH_URLS` | `https://wroclaw.pasport.org.ua/solutions/e-queue` | які сторінки стежити (через кому — можна кілька) |
| `HEARTBEAT_HOURS` | `24` | раз на скільки годин слати «я живий»; `0` — вимкнути |
| `ERROR_ALERT_AFTER` | `3` | скільки збоїв поспіль до сповіщення «не можу перевірити» |
| `COOLDOWN_MIN` | `30` | пауза між повторними нагадуваннями «є місця» |
| `USE_BROWSER` | `0` | `0` — простий HTTP (рекомендовано, сайт SSR), `1` — браузер (JS) |
| `UNAVAILABLE_PHRASES` | `всі місця зайняті\|запис відсутній` | фрази «місць немає», розділені `\|` |
| `AVAILABLE_MARKERS` | `queueform\|qlogickform` | ознаки активної форми запису (= є місця) |
| `PAGE_MARKERS` | `черг\|запис\|e-queue` | ознаки, що сторінка завантажилась |

> Усі ці змінні **необовʼязкові** — якщо не задавати, беруться робочі значення за
> замовчуванням. Порожні значення теж безпечні (скрипт підставить дефолт).

### 2.4. Перевір, що працює

Репозиторій → вкладка **Actions** → workflow **pasport-queue-watch** →
кнопка **Run workflow** (це `workflow_dispatch`). Подивись у логах, чи сторінка
завантажилась і що визначив скрипт. Далі він запускатиметься сам за розкладом.

> ℹ️ GitHub автоматично **вимикає** заплановані workflow, якщо в репозиторії
> 60 днів немає активності. Іноді заходь і роби «Run workflow» або невеликий коміт.

## Частина 3. Локальний запуск (альтернатива)

### Windows (PowerShell)

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
# python -m playwright install chromium   # лише якщо ставиш USE_BROWSER=1

# Задай змінні на поточну сесію:
$env:TELEGRAM_BOT_TOKEN="твій_токен"
$env:TELEGRAM_CHAT_ID="твій_chat_id"
$env:LOOP_SECONDS="300"   # перевіряти кожні 5 хв; або прибери для одного прогону

python watch.py
```

### macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
# python -m playwright install chromium   # лише якщо ставиш USE_BROWSER=1

export TELEGRAM_BOT_TOKEN="твій_токен"
export TELEGRAM_CHAT_ID="твій_chat_id"
export LOOP_SECONDS=300

python watch.py
```

У режимі `LOOP_SECONDS>0` скрипт працює нескінченно, доки не зупиниш (Ctrl+C).
Компʼютер має бути ввімкнений. Для цілодобової роботи зручніший GitHub Actions або VPS.

## Тонке налаштування (зазвичай не треба)

Дефолти вже підлаштовані під реальний сайт (перевірено 07.2026: Вроцлав показував
«Наразі всі місця зайняті», Краків — активну форму запису). Чіпай ці змінні, лише
якщо сайт змінить верстку:

- `UNAVAILABLE_PHRASES` — фрази «місць немає» через `|` (регістр не важливий).
- `AVAILABLE_MARKERS` — маркери активної форми запису через `|`.
- `PAGE_MARKERS` — ознаки, що сторінка взагалі завантажилась.

Якщо в логах постійно стан **⚠️ помилка** з причиною *«немає маркерів сторінки»* —
можливо, спрацьовує Cloudflare. Спробуй `USE_BROWSER=1` (браузер частіше проходить
челендж; тоді розкоментуй встановлення Chromium і став `USE_BROWSER=1` як variable).

## Часті питання

**Чому саме 5 хвилин, а не щосекунди?**
GitHub не запускає cron частіше ніж раз на ~5 хв, і це чесно щодо сервера сайту.
Надто часті запити можуть призвести до блокування. 5–10 хв — розумний баланс.

**Скрипт записує мене в чергу автоматично?**
Ні. Він лише **повідомляє**, що місця зʼявились. Записуватись треба самому й швидко —
такі слоти часто зникають за хвилини.

**Чи законно так робити?**
Це звичайний періодичний перегляд публічної сторінки для особистого користування.
Тримай інтервал помірним і не створюй навантаження на сервіс.
